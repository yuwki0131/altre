# Tauri GUI React UI 実装

## タスク概要
React を用いた Tauri GUI のフロントエンドを実装し、バックエンドの API と連携させる。

## 目的
- エディタ画面（バッファ、ミニバッファ、モードライン）の基本 UI を React コンポーネントとして構築する。
- キー入力イベントを捕捉し、バックエンドへキーシーケンスとして送信する。
- Pull 型スナップショット API を利用して描画を更新できるようにする。

## 成果物
- `frontend/react/` 配下にページ／コンポーネント構成、状態管理ロジック（例: Zustand/Redux/Context）を実装。
- キーイベントとマウスホイール（必要最低限）をバックエンドに橋渡しするハンドラ。
- 初期表示と状態更新を確認できる最小 UI（テキスト表示、ミニバッファ、メッセージエリア）。
- ESLint/Prettier など開発用設定（可能であれば）。

## 前提条件
- Tauri プロジェクト構成整備タスクで React 雛形が整っている。
- バックエンド連携 API が利用可能であること。
- キーシーケンスのシリアライズ仕様が設計タスクで定義済み。

## 完了条件
- [x] アプリ起動時に React UI が表示され、バッファ内容を取得して描画できる。
- [x] キー入力（文字入力、カーソル移動、保存）がバックエンド経由で反映される。※保存はバックエンド未実装のため fallback メッセージ表示
- [x] ミニバッファやメッセージ表示がバックエンドからのデータで更新される。
- [x] 主要な UI コンポーネントについて Storybook などの補助ツールが不要である理由（小規模構成・将来検討）を記録。
