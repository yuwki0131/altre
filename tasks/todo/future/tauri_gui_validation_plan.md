# Tauri GUI 動作確認計画

## タスク概要
Tauri GUI の基本機能を検証するテスト計画を策定し、手動/自動テスト手順とログ確認方法を定義する。

## 目的
- MVP 相当の編集フロー（ファイルオープン、文字入力、保存、終了）を GUI で検証できるようにする。
- NixOS + Wayland 環境での起動手順、制約、ワークアラウンドを整理する。
- 将来的な自動化（tauri-driver 等）の導入余地を検討する。

## 成果物
- `docs/design/tauri_gui_validation.md`（新規）にテストシナリオ、確認項目、ログ取得手順を記載。
- 手動テストチェックリスト（最新操作記録を `log.md` に追記）。
- CLI オプションで指定するデバッグログの確認方法と解析例。

## 前提条件
- バックエンド連携実装および React UI が動作するビルドが存在する。
- デバッグログが `~/.altre-log/debug.log` に出力される実装が完了している。

## 完了条件
- [x] GUI 起動から終了までの手順が記載され、NixOS 前提の注意点が明文化されている。
- [x] MVP 操作のテストケースと期待結果が列挙されている。
- [x] ログ解析手順（JSON Lines 等）が記録され、既知の制限や今後の課題が整理されている。
- [x] チェックリストに沿った検証結果テンプレートが提供されている（log.md 追記をテンプレートとする）。
